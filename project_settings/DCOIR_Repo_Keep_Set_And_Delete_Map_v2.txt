DCOIR Repo Keep Set and Delete Map v2

Purpose
Use this attachment as a final-cleanup planning map for malwaredevil/dcoir-collector. It does not authorize deletion by itself. Deletions are deferred to T99 and require current snapshot review, Airtable registry checks, source-authority review, and final validation.

Always keep unless current authority changes
- .github/ issue templates, workflows, and GitHub repo support files.
- .gitignore and other repository metadata files that belong in the working tree.
- AGENTS.md when present and current.
- README.md and current folder README surfaces.
- Current governed source files required by active Airtable plans or Repo Surface Registry.
- project_sources/DCOIR_Collector.ps1 unless a source-authority review changes the collector source role.
- project_sources prompt-pack / Gemini source folders that active plan state or source review keeps current.
- dcoir_skills/ current governed helper-skill source folders.
- dcoir_skills/project_discovery_contract.json when still used for repo source-role discovery.
- dcoir_skills/skill_parity_manifest.json and dcoir_skills/skill_parity_summary.md when current.
- knowledge/task_memory/ canonical reusable task-memory bank when still referenced by preflight/source review.
- project_settings/ current project settings mirrors when intentionally retained.
- release_notes/ current release notes and promoted delivery history.
- supporting_assets/ retained assets still referenced by current workflows.

Detached from normal startup; retain as fallback/promoted history until T99 decision
- project_sources/CP-01_DCOIR_Version_Manifest.txt
- project_sources/CP-02_DCOIR_Change_Log.txt
- project_sources/LOG-01_DCOIR_Todo_Log.txt
- project_sources/LOG-01_DCOIR_Todo_Index.txt
- project_sources/todo/01_Active_Now.txt
These are replaced for normal startup by Airtable Governance Control Plane, Queue Control, Work Items, Plans, Plan Tasks, Session Checkpoints, Operator Preferences, Repo Surface Registry, Repo File Coverage Detail, and Retained Repo Manifest.

Treat as historical unless explicitly reauthorized
- GitHub todo files and old active-now files retained as retired migration surfaces.
- old queue references superseded by Airtable Queue Control / Work Items / Plans / Plan Tasks.
- old helper-memory markdown under dcoir_skill_memory/ after matching Airtable memory coverage is verified.
- transition notes that have been superseded by a later promoted closeout.

T99 deletion candidates, not immediate deletions
- dcoir_skill_memory/ and related GitHub markdown helper-memory files after Airtable coverage check.
- duplicate or obsolete readable-source files not marked current by Airtable registry/source review.
- transitional bundles or generated zips that are no longer retained supporting assets.
- runtime residue such as __pycache__/, *.pyc, .DS_Store, temporary folders, or local scratch files.
- obsolete workflow backups after Airtable registry/source-authority review classifies them safe to remove.

T99 final cleanup method
1. Operator uploads latest repo snapshot.
2. Assistant verifies snapshot root shape and excludes .git.
3. Assistant reads Airtable Governance Control Plane, Queue Control, Work Items, Plans, Plan Tasks, Repo Surface Registry, Repo File Coverage Detail, Retained Repo Manifest, Skill State Registry, and active plan state.
4. Assistant reads GitHub CP/source files only when retained source-role or promoted-history comparison requires them.
5. Assistant builds a clean final repo working tree.
6. Assistant packages one full replacement ZIP with no wrapper root and no .git directory.
7. Operator preserves local .git, deletes/replaces the rest of the working tree, extracts the ZIP, inspects in GitHub Desktop, commits, and pushes.
8. Assistant verifies remote GitHub readback before final completion.

Hard stop
Do not remove a file or folder solely because it looks stale. Every deletion must have a current authority basis and no remaining live dependency.
