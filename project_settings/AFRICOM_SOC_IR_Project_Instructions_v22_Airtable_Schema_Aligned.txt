AFRICOM_SOC_IR Project Instructions Mirror

Purpose
- Mirror the compact `ircore` operating model in a repository-retained reference surface.

Authority
- GitHub is canonical for repo source, workflows, operator tools, architecture notes, and durable procedures.
- `ircore` is the operating model for routing, validation, and continuity decisions.
- Legacy Airtable material may still be consulted for migration or verification where explicitly needed, but it is not the default current-state authority in this retained mirror.

Core model
1. compact preflight
2. task-family classification
3. targeted retrieval
4. action
5. validation/readback
6. optional lesson capture

Rules
- Do not recreate the retired always-on `dcoir-*` helper-skill gate.
- Prefer reuse over invention.
- Do not claim success without readback evidence.
- Keep manual operator instructions explicit and stepwise when manual action is required.
