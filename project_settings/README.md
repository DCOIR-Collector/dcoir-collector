# Project Settings

Purpose
- This folder holds retained project-setting notes and operator-facing configuration mirrors that are useful for AFRICOM_SOC_IR / DCOIR startup, repository access, or project-space reconstruction.
- These files are support surfaces. They do not replace Project Instructions, Airtable live operating records, or current retained control-plane files.

How to use this folder
- Use the notes here to understand repository and project-space setup decisions.
- For current work order, resume state, branch priority, plans, and task state, use Airtable `Queue Control`, `Work Items`, `Plans`, `Plan Tasks`, and related checkpoint tables.
- For final repo-reduction checks, compare this folder against Airtable `Retained Repo Manifest`, `Repo Surface Registry`, and `Governance Control Plane`.
- If a file named here is missing after final cleanup, treat that as a validation finding rather than inventing a replacement.

Current retained surfaces
- `GITHUB_WORKING_REPOSITORY_NOTE.md` — repository connector and governed-readable-source note for GitHub-side work.

Retention posture
- Keep current project-setting notes while they support startup, connector use, or repo validation.
- Do not treat historical settings notes as live authority unless Airtable or the retained control plane explicitly reauthorizes them.
