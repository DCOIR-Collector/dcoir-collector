# GitHub Working Repository Note

Use the GitHub connector against the repository `malwaredevil/dcoir-collector` for DCOIR repo-side read, update, branch, PR, and review work.

Rule:
- GitHub is the governed readable source for retained repository files, helper-skill source, release history, and promoted workflow decisions.
- Airtable is the live operating authority for queue order, branch priority, active execution order, plans, plan tasks, session carry-forward, operator preferences, validation catalog state, and stateful helper-skill durable memory where tables exist.
- Do not keep duplicate ordinary editable readable text files in both GitHub and Project space.
- Project space is the bootstrap and applied runtime anchor.
- Project Instructions remain the first startup anchor when supplied by the operator.
- When governed readable files are needed, use the GitHub connector against `malwaredevil/dcoir-collector` while those files remain retained and current.
- If a referenced GitHub file is absent after final repo reduction, use Airtable `Governance Control Plane`, `Repo Surface Registry`, `Retained Repo Manifest`, `Queue Control`, `Work Items`, `Plans`, and `Plan Tasks` to resolve live operating state and record the missing retained source as a validation finding.
- Do not treat repo contents alone as live operating authority unless the governed project model explicitly says so.
