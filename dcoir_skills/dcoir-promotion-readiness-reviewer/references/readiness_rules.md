# Readiness Rules

A candidate is not ready when authority is unclear, deep regression is missing for testable artifacts, required release instructions are missing, or downstream refresh work is unresolved.

For batched skill-update waves, not-ready also includes missing per-skill regression coverage or a missing outer delivery bundle.
For GitHub Desktop manual repo-update deliveries, not-ready also includes a missing repo-update bundle, missing required delivery instructions, or missing suggested commit-summary guidance when the current workflow requires it.
