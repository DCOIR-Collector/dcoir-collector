PP-09 Gemini Enterprise Agent Designer Generator Workflow v1.0.0

PURPOSE
- Approved workflow for designing, validating, storing, and packaging Gemini parent-agent and sub-agent field sets for the DCOIR line.
- Shift the default Gemini shipment path from on-the-fly generation to stored-source compilation.

AUTHORITY ORDER
1. Project Instructions
2. CP-01
3. CP-02
4. PP-01 through PP-07
5. DOC-04 and DOC-10
6. PP-10 for bounded Gemini design work
7. PP-08 as parity reference only
8. DOC-07, generation-validation docs, and current logs as supporting guidance

NON-NEGOTIABLE RULES
- PP-01 through PP-07 remain design-time authority.
- Generated or edited outputs must be runtime-self-sufficient.
- Preserve evidence-first behavior, bounded confidence, command-lane separation, collector-awareness, and explicit next-step guidance.
- Generated or edited outputs must explicitly cover collector-artifact interpretation, collector pivoting, IOC-enrichment ownership, false-positive-aware security-product validation, and operator-state awareness.
- Reject filler endings and weak assistant chatter in analyst-facing outputs.

DEFAULT GEMINI SHIPMENT MODEL
- The default operator-facing Gemini bundle should be compiled from `project_sources/DCOIR_Gemini_Bundle_Source/`.
- Treat that folder as the governed editable runtime source tree for the parent agent, the sub-agents, and the approved knowledge attachment set.
- When requirements change, update the relevant files in that source tree, update the manifest or index when needed, and then compile the bundle.
- Do not default to regenerating the full Gemini bundle on the fly during ordinary packaging when the stored source tree is available.

WHEN GENERATION IS STILL ALLOWED
- Use generation or heavy redrafting only when one of these is true:
  - the operator explicitly asks for a redesign
  - the topology changes materially
  - a new sub-agent or major routing model is being proposed
  - the stored source tree needs a substantive rewrite before promotion
- After a generation or redesign round is accepted, promote the accepted runtime text into `project_sources/DCOIR_Gemini_Bundle_Source/` so future bundle downloads return to compile mode.

PROHIBITED SHORTCUTS
- Do not ship `knowledge/generated_agent_markdowns/` directly as the default operator-facing Gemini bundle.
- Do not package PP-09 or PP-10 themselves as substitutes for parent-agent or sub-agent runtime fields.
- Do not substitute governance docs, validation docs, or helper-workflow notes for the approved knowledge attachment set.
- Do not thin or summarize runtime agent files during compile-time packaging.

GEMINI ATTACHMENT BUDGET
- Hard ceiling for the office environment: 1000 KB per file, 2000 KB total per prompt.
- Safe target: 900 KB per file, 1800 KB total per prompt.
- Prefer chunk-first attachment sets.
- Do not assume a monolithic baseline report is the default uploadable surface.
- Scope clarification: this budget governs collector-emitted analyst-facing artifacts and default Gemini attachment sets built from them.
- It does not require shrinking, chunking, or thinning backbone deliverables such as DCOIR_Collector.ps1, run_DCOIR_Tests.ps1, PP-01 through PP-10, the full standalone master prompt assembly, or parent/sub-agent definition markdowns.

REQUIRED VALIDATION FOR GEMINI BUNDLES
- required files present
- required field sections present
- .txt suffix on operator-facing deliverables that may cross mail scanners
- approved knowledge attachment set present
- collector-artifact interpretation visibility
- explicit collector pivot behavior
- explicit IOC ownership
- false-positive-aware security-product behavior
- operator-state awareness and no filler
- compile-source conformance: ordinary shipment bundles must be traceable to `project_sources/DCOIR_Gemini_Bundle_Source/`

REQUIRED COMPILE WORKFLOW
1. Confirm the current control plane.
2. Confirm the stored runtime source tree under `project_sources/DCOIR_Gemini_Bundle_Source/`.
3. Edit the affected files directly in that tree.
4. Update the manifest and index when the version, topology, or required file set changed.
5. Run `project_sources/generation_validation/compile_dcoir_gemini_bundle.py`.
6. Validate the compiled output.
7. Ship the compiled zip.

END STATE
- Future Gemini bundle downloads should come from a governed stored-source and compile path.
- Design-time generation remains available for intentional redesign, but routine packaging should be compile, not improvisational generation.
