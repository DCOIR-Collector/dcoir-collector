PP-09 Gemini Enterprise Agent Designer Generator Workflow v1.0.0

PURPOSE
- Approved workflow for designing, validating, and packaging Gemini parent-agent and sub-agent field sets for the DCOIR line.
- Generate fresh standalone and Gemini deliverables from the current authoritative modular source line.

AUTHORITY ORDER
1. Project Instructions
2. CP-01
3. CP-02
4. PP-01 through PP-07
5. PP-08 as parity reference only
6. DOC-04, DOC-07, generation-validation docs, and current logs as supporting guidance

NON-NEGOTIABLE RULES
- PP-01 through PP-07 remain design-time authority.
- Generated outputs must be runtime-self-sufficient.
- Preserve evidence-first behavior, bounded confidence, command-lane separation, collector-awareness, and explicit next-step guidance.
- Generated outputs must explicitly cover collector-artifact interpretation, collector pivoting, IOC-enrichment ownership, false-positive-aware security-product validation, and operator-state awareness.
- Reject filler endings and weak “assistant chatter” in analyst-facing outputs.

GEMINI ATTACHMENT BUDGET
- Hard ceiling for the office environment: 1000 KB per file, 2000 KB total per prompt.
- Safe target: 900 KB per file, 1800 KB total per prompt.
- Prefer chunk-first attachment sets.
- Do not assume a monolithic baseline report is the default uploadable surface.

REQUIRED VALIDATION FOR GENERATED BUNDLES
- required files present
- required fields present
- .txt suffix on operator-facing deliverables that may cross mail scanners
- upload-budget conformance for default attachment sets
- collector-artifact interpretation visibility
- explicit collector pivot behavior
- explicit IOC ownership
- false-positive-aware security-product behavior
- SINGULAR TRIAGE COMMAND contract conformance
- operator-state awareness and no filler

REQUIRED GENERATION VERIFICATION
- Gemini bundle verification must check structure, required files, required field presence, attachment map completeness, and attachment-budget readiness.
- Standalone bundle verification must check full assembly from PP-01 through PP-07, required files, and .txt-safe packaging.
- Use the deterministic verification and QC artifacts under project_sources/generation_validation/ when they are in scope.

AUTOMATION POSTURE
- Automate what is feasible before live use:
  - bundle structure verification
  - field/section presence checks
  - upload-budget checks
  - fixture-driven prompt and artifact review checks where practical
- Do not overclaim full Gemini Enterprise UI automation when the platform path still requires manual operator setup.
- Treat the manual Enterprise UI run as a separate live-validation lane after repo-side verification passes.

PREFERRED OUTPUTS
- one standalone bundle
- one Gemini bundle
- one source-mapping artifact
- one verification/QC note
- one attachment map

END STATE
- Fresh Gemini and standalone bundles should be smaller, more deterministic, easier to quality-check, and easier to use in the operator’s real office Gemini environment.
