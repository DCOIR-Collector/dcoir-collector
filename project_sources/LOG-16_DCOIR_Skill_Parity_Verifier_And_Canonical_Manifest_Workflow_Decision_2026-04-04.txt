DCOIR Skill Parity Verifier And Canonical Manifest Workflow Decision - 2026-04-04

Current state id: 2026-04-04-skill-parity-v1

Decision
- Add a dedicated governed helper skill `dcoir-parity-verifier`.
- Treat `dcoir_skills/skill_parity_manifest.json` as the canonical machine-readable skill parity surface.
- Treat `dcoir_skills/skill_parity_summary.md` as the generated human-readable parity companion.
- Use per-file `sha256` values and a normalized tree hash as the primary parity evidence.
- Treat install or release zip hash as a secondary package or install check only.
- Prefer discovery of current skill trees and current manifest state over hard-coded file lists when parity work can read the current source directly.

Why
- Repo-versus-installed parity verification must move beyond version-only checks.
- README-only parity tracking is too fragile to be the primary source of truth.
- Zip hashes alone are too brittle because packaging metadata can change without content drift.
- The project is growing, and skills should minimize hard-coded project variables when current project state can be discovered instead.

Baseline posture
- The first seeded parity manifest in this branch is a mixed bootstrap built from the current installed skill tree plus the new parity-verifier skill.
- That baseline is useful for controlled startup and later comparison, but it is not yet a full repo-source-verified canonical baseline.
- The next bounded work item is a full repo-source rebaseline using `dcoir-parity-verifier`.

Required workflow
1. When a governed skill changes, refresh the canonical manifest.
2. Regenerate the human-readable summary from the manifest.
3. Package or refresh the install zip when needed and record its zip hash.
4. Before claiming parity, compare installed skill trees or install zips against the canonical manifest.
5. When a skill is removed, update the manifest and summary in the same governed change.
6. Use the parity results to support the broader helper-skill deep-dive and hard-coded-project-variable audit.
