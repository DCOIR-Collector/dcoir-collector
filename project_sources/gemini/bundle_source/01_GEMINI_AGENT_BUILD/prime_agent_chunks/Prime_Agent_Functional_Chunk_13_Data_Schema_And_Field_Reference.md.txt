### Data schema and field reference

```text
Use field references as guides, not guarantees. Prefer actual observed fields from the alert, query result, or uploaded artifact.

Common Elastic fields to consider when present:
1. @timestamp
2. event.kind, event.category, event.type, event.action, event.code, event.outcome
3. rule.name, rule.id, rule.description, signal.rule.name, kibana.alert.rule.name
4. host.name, host.id, host.hostname, host.ip, host.os.*
5. user.name, user.id, user.domain, user.email
6. agent.id, agent.name, agent.type, agent.version
7. process.name, process.executable, process.command_line, process.args, process.pid, process.parent.*, process.hash.*, process.code_signature.*
8. file.name, file.path, file.extension, file.hash.*, file.size, file.created, file.modified
9. registry.path, registry.key, registry.value, registry.data.strings
10. dll.name, dll.path, dll.hash.*
11. network.direction, network.transport, source.*, destination.*, dns.*, url.*, http.*, tls.*
12. threat.*, related.*, labels.*, tags.*

Field-use rules:
1. Use exact provided field names first.
2. If a field has both analyzed and keyword variants, choose the variant suitable for exact matching.
3. Avoid over-specific field names when the source evidence only supports a broader one.
4. If a query depends on a field not shown in evidence, state that dependency.
5. Do not hard-code a schema from memory when the current result shows a different schema.

CSV/XLSX and report fields:
1. Preserve original column names when parsing uploaded reports.
2. Do not rename or normalize columns silently when the exact wording matters.
3. If columns are missing, state the missing columns and how that affects confidence.
4. For USB reports, preserve fields needed for NIPR/SIPR separation, reporting window, SNOW-prefix classification, user/device identification, and correction tables.

Collector artifact fields:
1. Use collector report labels and sections as written.
2. Do not merge separate collector files into one source unless they were generated by the same run and scope is clear.
3. Preserve host, timestamp, collection mode, artifact path, and section names when interpreting output.
```
