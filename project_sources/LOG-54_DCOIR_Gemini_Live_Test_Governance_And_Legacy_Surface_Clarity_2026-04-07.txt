DCOIR Gemini Live-Test Governance And Legacy Surface Clarity - 2026-04-07

Current state id: 2026-04-07-gemini-live-test-governance-and-legacy-surface-clarity-v1

Purpose
- Record the governance hardening driven by the April 7 Gemini live-test session.
- Capture the immediate repo-hygiene interpretation for current-versus-legacy Gemini-readable surfaces.

What changed
- Added DOC-07 to preserve the specific April 7 live-test generation rules and the readable-surface role map for legacy Gemini material.
- Promoted detailed live-test findings into governed generation guidance instead of leaving them only in session notes.
- Clarified that the old Elastic triage agent set remains the verbosity and field-quality benchmark while the DCOIR-enhanced successor remains the current target.
- Clarified that generated/comparative markdown surfaces, extracted legacy bundles, failure-reference folders, and manual legacy captures each have different roles and should not be treated as interchangeable authority.

Why
- The project already had high-level viable-generation guidance, but the April 7 live test exposed more specific repeatable rules that needed to become durable.
- The repo contains several older, generated, comparative, and extracted Gemini surfaces whose roles were not yet explicit enough to prevent future confusion.

Live-test rules promoted now
- keep high-density Description and Instructions writing
- merge role or purpose content into one Description field for Agent Designer mapping when that is the cleaner generated shape
- standardize Instructions to the "You are ..." style by default
- include three starter prompts for the parent agent by default
- make collector-artifact triage and interpretation explicit
- make collector pivoting explicit when SIEM-native evidence leaves material gaps
- make IOC-enrichment ownership explicit
- add a false-positive-aware validation branch for known security products in lineage
- enforce singular-triage-command consistency
- require dedicated collector-artifact, collector-execution, IOC-enrichment, and repeated-session consistency validation

Legacy-surface interpretation preserved now
- `knowledge/generated_agent_markdowns/` is the readable generated working surface, not the design-time control plane
- `knowledge/comparative_reference_agent_markdowns/` is the readable comparative surface, not control-plane authority
- `project_sources/DCOIR_Gemini_Email_Build_Bundle_v1_1_0/` is structure-and-naming legacy reference only
- `project_sources/original_alert_triage_gemini_agents/` is the approved verbosity and field-quality benchmark
- `project_sources/original_collector_artifact_gemini_agents/` is a failure-reference surface
- `project_sources/legacy_reference_gemini_agents/` is the governed home for manual or curated legacy captures with ongoing reference value

What this does not yet claim
- It does not claim the broader legacy-inventory pass is complete.
- It does not claim every repo surface touching older Gemini material has already been normalized.
- It does preserve the current bounded interpretation so future work can continue without mistaking reference material for current build authority.

Best next move
- Continue with a bounded repo-hygiene inventory pass for all legacy-adjacent Gemini-readable surfaces before attempting broader moves, renames, or archival consolidation.
