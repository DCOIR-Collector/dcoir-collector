DCOIR Helper Skill Workflow And GitHub Source Rules

Purpose
- Record the evergreen workflow rules for DCOIR helper-skill development, Airtable-first queue authority, post-change regression, GitHub source rollout, and session-local buffer behavior.

Core standing rules

1. Airtable-only todo authority rule
- Airtable `Queue Control`, Airtable `Work Items`, and active Airtable `Plans` are the sole live todo authority for ordinary queue priority, active branch selection, resume-first order, and branch supersession.
- GitHub todo files are retired as live queue surfaces.
- GitHub still owns control-plane files, governed readable source, and promoted history.

2. Closed-loop memory-preflight rule
- Use `dcoir-memory-preflight` before high-friction execution and again after blocker recovery when the lesson may be reusable.

3. Session-local write-buffer rule
- Relevant helper skills may keep session-local buffer content until the next suitable grouped write or handoff point.

4. Mandatory post-change skill regression rule
- After every helper-skill create or update, pass the result through `dcoir-skill-regression-auditor` before treating the skill as ready.

5. Installed-skill marker verification rule
- For manual skill-install update flows, remove old `skill-marker:` lines from edited files, add one fresh marker to each edited file, and verify the expected marker in the installed skill before treating the install as safe for GitHub source sync, parity closure, or readiness claims.
- The skill editor is the primary truth surface for installed-skill verification when available.
- Assistant-side readback may lag and is secondary.

6. Updated-skill GitHub-source rollout rule
- Default updated-skill workflow:
  - build the updated skill package
  - manually install the skill update
  - verify installed-skill markers in the edited file set
  - update governed GitHub source or provide a GitHub Desktop manual repo-update bundle
  - run regression before readiness claims

7. GitHub promotion rule for queue changes
- Promote only important queue decisions, authority-model changes, and closeouts into GitHub.
- Do not recreate ordinary queue churn in GitHub todo files.
