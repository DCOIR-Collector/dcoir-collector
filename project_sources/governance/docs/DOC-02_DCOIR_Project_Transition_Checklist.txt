DCOIR Project Transition Checklist

Purpose
- Evergreen checklist for bounded repository updates.

Before applying an update
- confirm scope
- confirm the update belongs in one grouped change
- confirm active docs and workflows will not be left with broken references

When skill-related guidance changes
- validate the changed repo guidance directly
- do not require parity with a retired repo-side skill mirror

After applying the update
- re-read the affected files
- confirm active workflow and documentation references are still valid
- keep historical evidence cleanup out of scope unless explicitly included
