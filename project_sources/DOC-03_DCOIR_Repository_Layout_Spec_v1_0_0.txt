DCOIR Repository Layout Spec 1.1.4

Purpose
- This file defines the canonical GitHub-primary repository and Project-bootstrap layout for DCOIR.

Layout
- GitHub repository malwaredevil/dcoir-collector is the sole readable working-source location for governed text.
- Current readable governed text lives in:
  - project_sources/
  - knowledge/
  - project_settings/
  - release_notes/
- Retained binary or reference assets live in supporting_assets/.

Readable-source rules
- Collector runtime file:
  - project_sources/DCOIR_Collector.ps1
- Collector readable source set:
  - project_sources/collector_parts/DCOIR_Collector.01_Core_State_And_Utilities.ps1
  - project_sources/collector_parts/DCOIR_Collector.02_Baseline_Collection_And_Reports.ps1
  - project_sources/collector_parts/DCOIR_Collector.03A_Enrich_Session_State.ps1
  - project_sources/collector_parts/DCOIR_Collector.03B_Enrich_Actions_Review.ps1
  - project_sources/collector_parts/DCOIR_Collector.03C_Enrich_Actions_Retrieval.ps1
  - project_sources/collector_parts/DCOIR_Collector.04_Quick_Interface_And_Output.ps1
  - project_sources/collector_parts/DCOIR_Collector.05_Main_Entry.ps1
- Harness readable source:
  - project_sources/run_DCOIR_Tests.ps1
- Todo log and split todo structure:
  - project_sources/LOG-01_DCOIR_Todo_Log.txt
  - project_sources/LOG-01_DCOIR_Todo_Index.txt
  - project_sources/todo/*.txt
- Session handoff:
  - project_sources/LOG-03_DCOIR_Session_Handoff_Brief.txt
- Knowledge markdown in knowledge/ is the readable working source when present.
- Extracted readable source folders currently include:
  - project_sources/DCOIR_Gemini_Email_Build_Bundle_v1_1_0/
  - project_sources/original_alert_triage_gemini_agents/
  - project_sources/original_collector_artifact_gemini_agents/
  - knowledge/comparative_reference_agent_markdowns/
  - knowledge/generated_agent_markdowns/
  - knowledge/supporting_knowledge_docs/

Resume rules
- Resume from Project Instructions first.
- Then use the GitHub connector against malwaredevil/dcoir-collector.
- Read project_sources/CP-01_DCOIR_Version_Manifest.txt first and project_sources/CP-02_DCOIR_Change_Log.txt second.
- Treat GitHub as the working repository and review surface.
- Treat Project space as bootstrap/runtime anchor, not a second readable-source repository.
- For operator-side bulk file placement or binary and zip handling, GitHub Desktop on a local clone is the easiest approved path.
