DCOIR Repository Layout Spec 1.1.0

Purpose
- This file defines the canonical GitHub-primary repository and Project-bootstrap layout for DCOIR.

Layout
- GitHub repository malwaredevil/dcoir-collector is the sole readable working-source location for governed text.
- Current readable governed text lives in:
  - project_sources/
  - knowledge/
  - project_settings/
  - release_notes/
- Retained binary or reference assets live in supporting_assets/.

Project bootstrap layout
- Project space should contain bootstrap material only:
  - current Project Instructions text
  - release instructions
  - intentionally retained supporting assets when needed
- Do not place duplicate editable readable governed text files in Project space.

Readable-source rules
- Native readable script sources in GitHub are:
  - project_sources/DCOIR_Collector.ps1
  - project_sources/run_DCOIR_Tests.ps1
  - project_sources/run_DCOIR_Tests.cmd
- Knowledge markdown in knowledge/ is the readable working source when present.
- supporting_knowledge_docs.zip may remain a delivery/reference asset, but not the preferred editable readable source when extracted markdown exists.

Resume rules
- Resume from Project Instructions first.
- Then use the GitHub connector against malwaredevil/dcoir-collector.
- Read project_sources/CP-01_DCOIR_Version_Manifest.txt first and project_sources/CP-02_DCOIR_Change_Log.txt second.
- Treat GitHub as the working repository and review surface.
- Treat Project space as bootstrap/runtime anchor, not a second readable-source repository.
