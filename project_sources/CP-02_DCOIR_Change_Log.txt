[CURRENT]
Version: 3.9.22
Change class: local-only session-tracker memory model and continuity refresh

What changed
- Removed the GitHub-backed helper-memory branch from `dcoir-session-tracker` and made the local JSON file the only tracker working-state surface for this skill.
- Refreshed the tracker skill instructions and references so they now route cross-session continuity through exported handoff artifacts or governed Project-file promotion instead of `dcoir_skill_memory/dcoir-session-tracker/session_tracker_state.md`.
- Prepared deletion of `dcoir_skills/dcoir-session-tracker/references/github_memory_workflow.md` and `dcoir_skill_memory/dcoir-session-tracker/session_tracker_state.md` as part of the local-only model transition.
- Refreshed the active todo and continuity surfaces so they no longer describe GitHub-backed tracker memory as an open design branch.

Why
- The tracker now has a real inspectable local JSON state file, so the extra GitHub-backed branch for this skill became unnecessary redundancy.
- Cross-session continuity for this skill should come from exported handoff artifacts or governed Project-file promotion, not from a parallel helper-memory surface.
- The project should stop carrying contradictory language about whether tracker GitHub memory still exists.

Validation notes
- The bounded regression for the local-only model passed the tested path: capture item, inspect local file, ask what remains, trigger move-to-another-session behavior, and verify close-out output against the local file.
- The local file and regression artifacts proved the tracker can operate without a GitHub-backed helper-memory branch.
