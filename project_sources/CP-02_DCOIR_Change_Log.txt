DCOIR Change Log

[CURRENT]
Version: 4.0.8
Current state id: 2026-04-15-airtable-todo-authority-migration-v1
Change class: Airtable-only todo authority migration

What changed
- Promoted Airtable `Queue Control`, Airtable `Work Items`, and active Airtable `Plans` to the sole live todo authority for ordinary queue priority and resume order.
- Retired GitHub todo files as live queue surfaces without removing GitHub as the control plane or governed readable source.
- Updated the affected resume, authority, decision, memory-preflight, session-tracker, plan-tracker, and change-impact helper skills to stop relying on GitHub todo files for active queue priority.
- Updated workflow docs and handoff text so future sessions re-anchor correctly.

Why
- The prior split model kept pulling resume behavior back toward stale GitHub queue lines even when Airtable held the more current and intentionally prioritized branch order.
- Airtable was already the primary durable execution-state surface for plan and session continuity, so promoting it to the sole live todo authority removes the repeated priority conflict.
- GitHub remains valuable as the control plane, governed readable source, and promoted history, but it is no longer the right place for ordinary queue churn.

Validation notes
- Airtable queue-control cutover is implemented and the migration branch is complete.
- Collector hardening is now the active branch.
- Validate-on-push troubleshooting, reduced Airtable UI rendering during re-anchor, and GitHub-issues alignment are queued follow-ons.
- Retired GitHub todo files are preserved only as migration notes, not as live authority.
