Purpose
- This log records the decision to make `dcoir-session-resume` and `dcoir-memory-preflight` the mandatory first substantive-turn bootstrap pair for every new AFRICOM_SOC_IR / DCOIR session.
- It also records the bounded review and optimization proposal for the current AFRICOM_SOC_IR project instructions and the Project-space bootstrap pointer `CP-00_DCOIR_GitHub_Primary_Bootstrap.txt`.

Decision summary
- At the first substantive AFRICOM_SOC_IR / DCOIR turn of every new session, invoke `dcoir-session-resume` first and `dcoir-memory-preflight` second before other substantive project work unless the request is clearly outside DCOIR scope or blocked by safety.
- Keep `dcoir-session-resume` responsible for re-anchoring to Project Instructions, consuming `CP-00` when present, reading `CP-01` then `CP-02`, and consuming the explicit drift gate before trusting supporting continuity surfaces.
- Keep `dcoir-memory-preflight` responsible for the immediate post-resume lane classification so the session does not drift into execution before canonical memory is consulted.
- Treat this first-turn bootstrap pair as a prompt-level enforcement rule, not only as a skill-description preference.
- Preserve the GitHub-primary working-source rule and the current control-plane order instead of broadening bootstrap authority.
- Keep the Project-space `CP-00` bootstrap pointer lean and anchor-focused rather than turning it into a second editable readable source set.
