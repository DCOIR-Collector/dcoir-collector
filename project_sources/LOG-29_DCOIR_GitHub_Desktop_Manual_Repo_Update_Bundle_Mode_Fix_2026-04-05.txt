Purpose

Decision summary
- Treat the missing GitHub Desktop affected-path patch-bundle lane as a real helper-skill workflow defect, not as a one-off chat formatting mistake.
- Repair `dcoir-repo-packager` by adding a dedicated `github-desktop-manual-update` packaging mode.
- Require that mode to accept one or more affected repo-relative `--include-path` values.
- Require that mode to preserve those repo-relative paths exactly, emit no wrapper root, and avoid extra helper files inside the ZIP.
- Surface the suggested GitHub Desktop commit `Summary` in the report and response instead of adding a commit-summary text file to the ZIP.

- The operator uses GitHub Desktop for some governed repo-update sessions and expects a drag-and-drop patch-style ZIP containing only the affected repo-relative files.
- The preference for that delivery shape was already governed, but the existing packager only covered repo-layout and bootstrap bundle classes.
- Without this repair, ChatGPT can drift into copy-paste file-content delivery even when the settled workflow calls for a ZIP artifact.
