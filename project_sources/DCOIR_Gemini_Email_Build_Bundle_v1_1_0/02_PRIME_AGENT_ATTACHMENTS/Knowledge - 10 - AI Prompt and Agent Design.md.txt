## Operating model

- Maintain modular prompt-pack source files as the authoritative build set.
- Maintain one promoted combined analyst-facing master prompt from the validated modular set for current AI-assistant use.
- Keep the modular files and the combined master prompt behaviorally aligned.
- Use the approved Gemini generator workflow to derive future Gemini Enterprise Agent Designer outputs from the validated modular source files, the promoted combined master prompt, and companion knowledge files, not from ad hoc prompt copies.
- Treat the approved Gemini workflow as a bounded workflow source, not as a replacement for modular prompt-pack authority.
- Treat the promoted bounded Gemini design artifact as the current generated design reference for the workflow line, not as a replacement for modular prompt-pack authority or the control plane.
- If no live Gemini generator runtime exists yet, keep validation claims bounded to workflow-source conformance rather than claiming unperformed runtime testing.

## Current roadmap after first bounded Gemini design-artifact promotion

- Maintain modular prompt-pack files, the combined master prompt, the Gemini generator workflow, the promoted bounded Gemini design artifact, and supporting knowledge files aligned with the stable collector line.
- Re-test prompt, guardrail, Gemini-workflow, and Gemini-design-artifact behavior after any future stable script or workflow change.
- Future regenerated or extended Gemini design artifacts should still derive through PP-09 from current authoritative sources rather than ad hoc prompt copies.
