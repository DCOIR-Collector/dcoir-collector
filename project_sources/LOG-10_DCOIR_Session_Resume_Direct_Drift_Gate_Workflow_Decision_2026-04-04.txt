Purpose
- Record the approved 2026-04-04 workflow decisions for binding `dcoir-session-resume` directly to the explicit `dcoir-source-authority-auditor` drift gate.
- Prevent plausible but stale resume summaries from bypassing the same active-surface alignment checks already introduced for the enforcement workflow.

Approved durable workflow decisions
- `dcoir-session-resume` should read `CP-01` first and `CP-02` second, then consume the `dcoir-source-authority-auditor` drift gate before trusting supporting continuity surfaces such as `LOG-03`, `LOG-05`, `LOG-01_DCOIR_Todo_Log.txt`, `LOG-01_DCOIR_Todo_Index.txt`, and `todo/01_Active_Now.txt`.

Direct resume-gate sequence
1. Re-anchor to Project Instructions, then `CP-01`, then `CP-02`.
2. Invoke the `dcoir-source-authority-auditor` drift gate on the active continuity surface set.
3. If the gate returns `hard_stop_conflict`, stop and report the exact mismatch, the affected surfaces, and the smallest remediation set.
4. If the gate returns `proceed_bounded`, continue only with bounded claims and explicit missing-surface callouts.
5. If the gate returns `clear_to_proceed`, continue with the normal `dcoir-session-resume` output contract.
6. Do not let a superficially familiar work line override an explicit drift result.
